# Deployment Checklist

This repo deploys as two services:

- Netlify runs the Next.js frontend in `frontend/`.
- Render runs the FastAPI backend from `backend/Dockerfile`.

The cheapest durable setup is Netlify for the frontend, Render Free for the backend, and Neon Free for Postgres. Render Free Postgres is easy for a trial, but expires after 30 days, so use Neon if you want the database to survive longer.

## 1. Create The Database

Create a Neon Postgres project and copy its connection string.

Use the pooled or direct connection string as `DATABASE_URL`. It should look like:

```text
postgresql://user:password@host/database?sslmode=require
```

The backend accepts this through the `DATABASE_URL` environment variable.

## 2. Deploy The Backend On Render

Create a Render Blueprint from this repo, or create a Web Service manually with these settings:

```text
Runtime: Docker
Branch: main
Dockerfile path: ./backend/Dockerfile
Docker context: .
Health check path: /api/health
```

Set these Render environment variables:

```text
ENVIRONMENT=production
DATABASE_URL=<your Neon Postgres URL>
JWT_SECRET=<generated by Render or your own long random value>
DEMO_USER_EMAIL=demo@3x.local
DEMO_USER_PASSWORD=<generated by Render or your own secure value>
CORS_ORIGINS=["https://<your-netlify-site>.netlify.app"]
DEMO_MODE=true
EIA_API_KEY=
LLM_SCORER_PROVIDER=heuristic
ALLOW_REGISTRATION=false
RATE_LIMIT_ENABLED=true
```

After Render deploys, copy the backend URL. It will look like:

```text
https://market-speculation-api.onrender.com
```

## 3. Deploy The Frontend On Netlify

Create a Netlify site from this GitHub repo. The root `netlify.toml` already points Netlify at the frontend.

Confirm these Netlify settings:

```text
Base directory: frontend
Build command: npm run build
Publish directory: .next
```

Set these Netlify environment variables:

```text
API_INTERNAL_BASE_URL=https://<your-render-backend>.onrender.com/api
NEXT_PUBLIC_API_BASE_URL=/api/backend
NEXT_PUBLIC_WS_BASE_URL=wss://<your-render-backend>.onrender.com/ws
SERVER_AUTO_LOGIN=true
DEMO_USER_EMAIL=demo@3x.local
DEMO_USER_PASSWORD=<same value as Render>
```

Deploy the site.

## 4. Add A Domain

Use two hostnames:

```text
www.example.com -> Netlify frontend
api.example.com -> Render backend
```

After both custom domains work, update the env vars:

Render:

```text
CORS_ORIGINS=["https://www.example.com"]
```

Netlify:

```text
API_INTERNAL_BASE_URL=https://api.example.com/api
NEXT_PUBLIC_WS_BASE_URL=wss://api.example.com/ws
```

Redeploy Netlify and Render after changing env vars.

## 5. Updating The Live App

Use `main` as the deployment branch.

```text
make a change
test locally
commit to main
push to GitHub
```

Netlify and Render will redeploy automatically from `main`.

## Notes

- Render Free web services sleep after 15 minutes without traffic. The first request after sleep can take about a minute.
- The backend container uses Render's `PORT` environment variable when present.
- Keep real secrets in Netlify/Render dashboards, not in git.
- Power BI is optional. Leave the `POWER_BI_*` variables unset until you have real Microsoft credentials.
- Add Redis later if you want production-grade live streams and background workers.
